Install

needs a recent stable Rust toolchain
git clone https://github.com/smedjan/smedjan.git
cd smedjan
cargo build --release        # Metal (macOS / Apple Silicon) — the default
# NVIDIA:
cargo build --release --no-default-features --features cuda

On macOS the Metal backend is the default and needs no extra setup. For CUDA you need the toolkit (12.x). Checkpoints are portable across both backends.

// 02

Quick start

tokenizer → train → generate
smedjan tokenizer --input corpus.txt --vocab-size 8192 --output tokenizer.bin
smedjan prepare   --input corpus.txt --tokenizer tokenizer.bin --output train.bin
smedjan train \
  --dataset train.bin --tokenizer tokenizer.bin \
  --size medium --batch-size 4 --seq-len 256 \
  --steps 50000 --lr 3e-4 --checkpoint-dir checkpoints/

# resume (restores model + optimizer state + step)
smedjan train --dataset train.bin --tokenizer tokenizer.bin --size medium \
  --steps 50000 --resume checkpoints/state_25000.bin

# generate
smedjan generate --checkpoint checkpoints/final.bin --tokenizer tokenizer.bin \
  --prompt "The" --stream

Subcommand reference

the real CLI surface

tokenizer

Train a BPE tokenizer on a corpus.

prepare

Convert text to the binary training format.

process

Quality filtering and deduplication.

mix

Blend datasets with custom ratios.

train

Pre-train from scratch (add --teacher-checkpoint for distillation).

sft

Supervised fine-tuning on instruction data.

dpo / dpo-prepare

Direct Preference Optimization alignment.

eval

Benchmark across categories.

generate

Inference: sampling, --stream, --speculative.

quantize

Q4 / Q8 post-training quantization.

export-gguf

Emit llama.cpp-compatible files (f32, q8_0).

export-safetensors

safetensors export (zero-dependency).

Run smedjan <command> --help for exact flags. Honest status: safetensors import is F32-only today (bf16/f16 is a follow-up); GGUF export covers f32 and q8_0; RWKV and block-sparse mixers are experimental; faithful HF inference parity is on the roadmap.

// 04

Core concepts

what you're actually training

Tokenizer

A from-scratch byte-pair encoder turns text into integer tokens and back. Train it once on your corpus; the same tokenizer is used for training and generation.

Model

A decoder-only transformer: RMSNorm, RoPE positions, Grouped-Query attention, SwiGLU MLP, weight-tied head. Sizes from 2M to 6.5B, or fully custom.

Checkpoints

A checkpoint holds the model weights plus optimizer state and step count, so training resumes exactly. The format is portable across the Metal and CUDA backends.

Generation

Autoregressive sampling with temperature, top-k / top-p (nucleus), and optional speculative decoding. Stream tokens as they are produced.

// 05

Fine-tuning & NLP tasks

from base model to instruction-follower

Pretraining gives you a base model that continues text. To make it follow instructions or match a style, fine-tune and align it — all with the same binary, no separate framework.

SFT — supervised fine-tuning

Teach the model to respond to instructions from JSONL prompt/response pairs with smedjan sft. This is how a base model becomes a chat or task model.

DPO — preference alignment

Nudge the model toward preferred answers using chosen/rejected pairs with smedjan dpo — lighter than RLHF, with no reward model to train.

Distillation

Train a small student from a larger teacher (KL + cross-entropy) via smedjan train --teacher-checkpoint, keeping quality while shrinking the model.

Quantize & export

Q4/Q8 quantization, GGUF export for llama.cpp, and safetensors for Hugging-Face interop: smedjan quantize, export-gguf, export-safetensors.

Every stage reads and writes the same checkpoint format, so you can chain pretrain → SFT → DPO → quantize → export without ever leaving Smedjan.
